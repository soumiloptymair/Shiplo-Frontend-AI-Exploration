| Split Shipment modal                  | `pages/shipments/split-shipment-modal`                               | Figma node `23008-101209` + workflow `27432-68529` (file `unEpC0FcuWKbB5yO1m7OyX`). Fixed 1324×812 dialog (caps at `95vh` / `calc(100vw-2rem)` so it stays stable while dropdowns or cards open). Subheader holds the edit copy on the left and `Add Shipment` + `Reset` on the right; Reset is disabled until the user dirties the recommendation. Cards area wraps 2-per-row and scrolls vertically when N≥3. Each card's warehouse dropdown is subtractive — warehouses chosen by other cards are removed from this card's options (own selection always retained). Trash icon is always rendered; disabled with a "At least 2 shipments are required" tooltip when `shipments().length <= 2`. Qty column header is right-aligned to sit above the cell values. Left 320 px column = 3 recommendation presets (Rec 1 selected by default, bg `#e0f5f0` + 2 px `#008572` border) + divider + "Manual split". Right area on `#f6f9fb` shows the active recommendation as 472 px shipment cards (warehouse picker + Product/Qty table with row checkboxes + per-card action menu) plus an "Add Shipment" pill. Rec 1 routes Shipment 2 to `PA Fulfillment Center`; whenever a bucket ships from a non-original warehouse an orange impact-preview banner is shown above the cards (delivery-time + cost copy). When a row is checked the Qty cell becomes an editable `qty / max` stepper so partial moves are possible. The action menu shows `N products, M items selected`; for 2 shipments it's "Move to Shipment N", for 3+ it becomes a "Move to" dropdown listing every other shipment. Removing an extra shipment opens a dark "Delete Shipment? · No · Delete" popover (red Delete) anchored to the trash icon. Footer Cancel + teal Split Shipment (disabled if any shipment is empty or <2 shipments). Confirm emits a `SplitConfirmPayload`; the panel forwards it to `ShipmentService.splitShipment(originalId, buckets)` which replaces the original grid row with N "Pending" rows labelled `${shipmentId} - 1 … N`, computes per-bucket `value` from `qty × unitValue`, selects the new rows so the grid highlights them, closes the side panel, and fires the global toast `Shipment ${shipmentId} split successfully`. |
